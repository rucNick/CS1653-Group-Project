2. Design Proposal

"CypherSpace"

Is a secure message posting board, where users are able to create, edit, or delete posts. These posts are then publicly displayed for other users to read. CypherSpace also contains an additional VIP level, in which only select users can participate. VIP users may read all the posts including the normal ones, but normal users may not access VIP posts. There is an additional third permission level for this application: Administrator level. Administrators are not only allowed to view all posts, but they can also delete posts made by other users as well. Additionally, administrators are the ones who control who gets VIP access and may revoke said access if deemed necessary. Besides these proposed permission levels, a ground level “guest” may read normal user’s posts across any board but may not participate in any way.

There are different message boards in which users may choose to partake in, depending on their area of interest. The user will be free to select which board is currently in view. Messages are saved on a separate database, allowing for message persistence after a user has closed the page. User authentication is handled via login, and once it has been achieved the logged in state will persist allowing the user to create multiple posts without having to re-authenticate.

Given this, the authentication server must handle user registration and authentication, as well as deal with the permission level for each user. Usernames and passwords are encrypted to provide protection against unauthorized entry. Also, encryption is also applied to the messages to ensure that each message only reaches those with permission to view it. The resource server must be able to discern which user should be provided with the normal messages, and which can view the VIP section. Both normal and VIP posts are stored on a separate database.

3. Security Properties

  1. Message Manipulability:

    Depending on the user's permissions level, they can have various actions. For example, a basic user is only able to manipulate their own messages (edit/delete). Hence a basic user should not be able to manipulate anyone else's messages without having higher permissions.

  2. Encrypted Login Data:

    When a user logs into the message board it is important for their login details to be encrypted before it is sent to the server to be validated. This prevents any malicious attackers from gaining access to the user’s account as they will not be able to get the login details.

  3. Message Accessibility:

    By default, there are some messages for the public and can be seen by all users, even those considered guests that may not have an account. At the same time there are also VIP messages that cannot be seen by the public and need special access/permissions. Which is why there is a need to properly validate a user’s permissions/access and give proper secure access.

  4. Encrypted Message Data:

    Since there are non-public VIP messages it is important for those to stay confidential to the general public. To ensure confidentiality the messages must be encrypted when VIP message data is getting sent between the message board and the various servers. So that no user below a VIP can access them.

  5. User Authentication:

    By any small chance a user’s information is leaked or gained by a 3rd party it is important for a 2nd Factor authentication to be put in place. This will validate the identity of the user and make sure that the party using the message board is the user themselves.

  6. Key Distribution:

    Implementing a secure way to distribute keys among the system is a necessity. This guarantees that data is properly encrypted and can also be decrypted while maintaining confidentiality. But if the key is not securely distributed an attacker can gain access decrypt or possibly encrypt malicious data into the system.

  7. Audit Logging:

    A log that tracks various user activities from messages, logins, etc. This helps with detecting any unauthorized activity or malicious intent and helps hold various bad actors accountable.

  8. Data Integrity:

    Ensure that when a message is posted it is not possible to be changed unless an administrator or the original poster wants it to. This ensures that all the posts on the board are genuine.

  9. Inactive Session Management:

    Depending on how long the message board has been opened by a user, the board should automatically log out the user depending on the length of inactivity of the user. This ensures that no physical outside force that is not the user to gain access to the message board. It also prevents various things such as session hijacking.

  10. Access Removal:

    If a user’s status changes from VIP to a normal user, they must no longer have access to any of the VIP messages/boards. Hence it must be ensured that privileges are properly removed and that they no longer have access.

  11. Encrypted Backups:

    Important for the message board to have backups of the data in case any of the servers run into any problems with the authenticity of the data or potential data corruption/loss. Hence it is necessary for the server to maintain the data in an encrypted way so that data can be recovered.

  12. Secure Account Recovery:

    Having a proper way for a user to securely recover their account if they have forgotten their logins. While also ensuring that they identify themselves, so another individual does not gain access to another account. This can be done by using security questions, text message or email verification.

  13. Input Validation:

    Ensures that all input fields in the system have proper input validation and prevents injections and any other malicious inputs. So that an attacker may not abuse any form of inputs to attack the system.

  14. Login Attempts:

    Limit the number of times a user can attempt to login to an account. This prevents any attacker from trying to secure access into the system through another’s account, especially prevents brute force.

  15. Bot Detection:

    Implementing a form of bot-detection such as CAPTCHAs will ensure that message boards are safe from automated spam. This also helps to make sure that all users can be held accountable for their actions as they are all legitimate. So, implementing a form of CAPTCHA during initial use will potentially help from any automated malicious intents.

  16. Virus Protection:

    As users post on the message board there may be different files that may be distributed. To guarantee the safety of other users it is important for files to be scanned for malware and must be done so in a secure environment. This helps ensure that the reputation of the board can be seen as safe.

  17. Complete Mediation:

    Every access to the resource server will be checked without relying on cached data. Every request to post, edit, or view content on the platform will be strictly validated against the defined permissions each time.

  18. Fail-safe defaults:

    Access to restricted (VIP) content is determined by the permission level of the user, instead of acting through exclusions.

  19. Least Privilege:

    Users with non-VIP access only have the necessary permissions to allow for participation on CypherSpace, and no additional actions outside of their planned scope are permitted.
